The FSM

And its properties

FSM

FSM is an abbreviation for many things. For us, it’s not the “Finite State Machine”, but actually it stands for “Full Scale Machine”. It is the top-most tree in the bot, whose name and id is equals with the project name.

Once you started a new project, you are entering its first tree. You will actually see only a root - the node with the  symbol. The name of the tree (to the left, under Trees) is a random GUID.

You’ll need to do to things:

  1. Rename your tree as the same name of the project. If the project is called banking bot, click on the root node, and change its title to match the project’s name
  2. Click on the root node, then on the properties icon  that will appear on the right. Now you’ll need to enter properties of the root

The root should have one child, and it is not really a true node. It only links to the real root of the tree, its single child. But, its properties are applied to the whole tree, its subtrees, its processes - as said, the FSM. Here are the properties names and explanation. It is advisable to take them from one of the sample trees that comes with the system.

Properties

resetMemory: if 1, it clears the memory of the process when the FSM is started, meaning that anything saved in the database previously is lost

channels: name of client. Currently, chatsim, alexa and facebook are supported

nlu: properties for the nlu of the tree. It is an object whose keys are the language codes. For example:
"nlu": {

    "he": {

      "engine": "wit",

      "accessToken": "7UVWOHBGLQ7VCQQRTVOIFREXUSNN6NIF"

    },

    "en": {

      "engine": "wit",

      "accessToken": "KDIYTFNWYPB72MUXSWV7YW3FF7MPLI5I"

    }

  },

facebook: properties specific to the facebook channel. Currently it has:

        validationToken: needed to hook the facebook app to the Servo bot

        accessToken: the token for the facebook app

defaultLang: the default 2-letter language code

debug-log: if non-empty, will be printed for every node that doesn't have its own debug-log. Can contain a <%= %> template
